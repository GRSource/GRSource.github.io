  • 1.建仓pod repo add 仓库名 仓库github地址

    • 2.创建工程：pod lib create YourFrameworkName，添加文件到Classes后要pod update，然后打好tag并提交项目github地址(跟仓库地址不一样)

  • 要发布到Cocoapods官方，只需要在前面私有库的基础上执行pod trunk push pod_test.podspec，然后更新本地pods仓库pod setup

# 本地检查pod spec合法性：pod lib lint --allow-warnings --verbose --sources='https://github.com/iOSAour/ModularizationPod.git,https://github.com/CocoaPods/Specs.git'

# 发布：pod trunk push NAME.podspec --allow-warnings

# babel

# @babel/core

  • babel的核心api

# @babel/cli

  • 这是一个通过命令对js文件进行转换的工具

# @babel/preset-env

  • 指定转换的工作环境

    • 之前babel通过几个预设来确定babel如何进行转换，比如babel-preset-es2015是将ES6转成ES5。但是如果浏览器已经支持了该语法还是会进行了转换。@babel/preset-env很好的解决了这一点，它的工作方式是允许你指定环境的同时，仅转换未支持的功能。

# @babel/polyfill(deprecated)

  • 相当于一个填充，因为babel本身只支持转换箭头函数、结构赋值这些语法糖类的语法，而一些新的API或者Promise函数等是无法转换的。@babel/polyfill就是解决这个问题的。但是在babel7.4之后遗弃了，直接使用import "core-js/stable"和import "regenerator-runtime/runtime"

# 解析Crash

# 获取crash文件的UUID：grep *crash

# 获取.dSYM的UUID：dwarfdump --uuid app.dSYM

# 获取主程序的UUID：dwarfdump --uuid <主程序文件路径>

# 获取symbolicatecrash文件：find /Applications/Xcode.app -name symbolicatecrash -type f

# 过程

  • 拿到crash文件，拿到dSYM文件，对比他们的UUID是否一致，一致的话进行下一步，不一致无法解析

    • 将crash文件、symbolicatecrash文件、dSYM文件放置同目录

      • ./symbolicatecrash ./demo.crash ./demo123123.app.dSYM > SymbolCrash.crash

        • 如果报错“Error: "DEVELOPER_DIR" is not defined at ./symbolicatecrash line 69.”，就执行：export DEVELOPER_DIR=/Applications/XCode.app/Contents/Developer
        • SymbolCrash.crash就是符号化过后的文件，可以看到crash函数

# 编译程序

# 预处理clang++ -E test.cpp -o test.i

这一阶段的过程主要是对包含源代码的文本文件进行处理。主要的处理内容就是将在源文件中包含的头文件加载到文件中，替换宏定义等等。最终生成的文件是以 . i .i .i结尾的文本文件。

  • 编译clang++ -S test.i -o test.s

    这个阶段是将预处理得到的文件经由编译器变成汇编语言。汇编语言中的每一条语句都以一种标准的文本格式准确的描述一条地址机器指令。在这个阶段生成的文件才是机器相关的代码。这个阶段生成以 . s .s .s结尾的汇编文本文件。

    • 汇编clang++ -c test.s -o test.o

      这个阶段就是将上一步得到的汇编文件汇编成机器指令，从而把这些指令打包成为一种可重定向的目标程序格式。这个时候生成以 . o ​ .o​ .o​结尾的二进制文件。

      • 链接clang++ test.o -o test.exe

        这个阶段主要是处理在文件中调用了系统库例如 c i n cin cin等函数，那么就需要将系统库中的 c i n . o cin.o cin.o合并到我们生成的 t e s t . o test.o test.o中。生成可执行的以结尾的 . e x e .exe .exe二进制文件。
